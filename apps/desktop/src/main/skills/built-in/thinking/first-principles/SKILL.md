# 제1원리 분해기

## 언제 사용하나요?
- 새로운 제품이나 기능을 설계할 때
- 복잡한 문제에 막혔을 때
- 기존 해법이 지나치게 복잡해 보일 때
- 가정을 도전해야 할 때
- 새 프로젝트나 이니셔티브를 시작할 때

## 프로세스

### 1단계: 가정 식별
질문: "당연하다고 믿고 있지만, 사실이 아닐 수 있는 것은?"
현재 접근법에 내포된 모든 가정을 나열합니다.

### 2단계: 원자 단위로 분해
각 가정에 대해 질문: "여기서 가장 근본적인 진실은 무엇인가?"
더 이상 분해할 수 없는 바닥(bedrock) 사실에 도달할 때까지 "왜?"를 반복합니다.

### 3단계: 진실에서 재구축
검증된 근본 사실만을 출발점으로:
"핵심 필요를 충족하는 가장 단순한 해법은?"

## 대화 흐름

Usan이 이 스킬을 실행할 때:

1. **문제 명확화** (최대 1~2개 질문)
2. **가정 도출** — 당연하게 받아들이고 있는 것 나열
3. **근본 사실로 분해** — 원자적 진리 제시
4. **해법 재구축** — 바닥부터 새로 조립
5. **비교** — 기존 접근법과의 차이 설명

## 출력 형식

```
문제: [서술된 문제]

식별된 가정:
1. [가정] → 도전: [이것이 틀릴 수 있는 이유]
2. [가정] → 도전: [이것이 틀릴 수 있는 이유]

근본 진리:
• [바닥 사실 1]
• [바닥 사실 2]
• [바닥 사실 3]

재구축된 해법:
[근본 사실만으로 만든 새로운 접근법]

기존 접근법과의 차이:
[통상적인 접근법과 어떻게 다른지]
```

## 예시 트리거
- "우리 고객 소통 문제를 제1원리로 분해해줘"
- "[X]를 처음부터 다시 생각하고 싶어"
- "[문제]에 대해 우리가 잘못 가정하고 있는 건 뭐야?"

## 연계 스킬

다음 스킬과 함께 사용하면 효과가 극대화됩니다:
- **pre-mortem** — 재구축된 해법의 실패를 미리 상상하여 스트레스 테스트
- **decision-trees** — 재구축된 선택지들의 기대값 분석
- **business-model-canvas** — 근본 진리 기반으로 비즈니스 모델 재설계
