# 의사결정 트리 — 구조화된 의사결정

의사결정 트리 분석: 확률과 기대값을 사용하여 의사결정을 시각적으로 구조화하는 도구입니다.

## 언제 사용하나요?

**적합한 경우:**
- 비즈니스 결정 (투자, 채용, 제품 출시)
- 개인 선택 (커리어, 이직, 구매)
- 트레이딩 및 투자 (포지션 크기, 진입/이탈)
- 운영 결정 (확장, 외주)
- 측정 가능한 결과가 있는 모든 상황

**적합하지 않은 경우:**
- 진정한 불확실성이 있는 결정 (블랙스완)
- 빠른 전술적 선택
- 순수하게 감정적/윤리적 질문

## 방법론

**의사결정 트리** = 트리 형태의 구조:
- **결정 노드** (사각형) — 당신의 행동
- **확률 노드** (원형) — 무작위 사건
- **종단 노드** (삼각형) — 최종 결과

**프로세스:**
1. **옵션 정의** — 가능한 모든 행동
2. **결과 정의** — 각 행동 이후 일어날 수 있는 일
3. **확률 추정** — 각 결과의 발생 가능성 (0~100%)
4. **가치 추정** — 각 결과의 효용/보상 (금액, 점수, 효용 단위)
5. **기대값(EV) 계산** — EV = 합계(확률 x 가치)
6. **선택** — 가장 높은 EV를 가진 옵션

## 공식

```
EV = 합계(확률_i x 가치_i)
```

**예시:**
- 결과 A: 70% 확률, +10만원 → 0.7 x 10만 = 7만원
- 결과 B: 30% 확률, -5만원 → 0.3 x (-5만) = -1.5만원
- **EV = 7만 + (-1.5만) = 5.5만원**

## 비즈니스 예시

**결정:** 신제품 출시 여부

```
제품 출시
├─ 성공 (40%) → +5억원
└─ 실패 (60%) → -2억원

EV = (0.4 x 5억) + (0.6 x -2억) = 2억 - 1.2억 = +8천만원

출시하지 않음
└─ EV = 0원
```

**결론:** 출시 (EV = +8천만원)가 미출시 (0원)보다 유리합니다.

## 사용자 워크플로

### 1. 구조화
다음을 확인합니다:
- 가능한 행동 옵션은?
- 가능한 결과는?
- 각 결과의 가치/효용은?
- 가치를 어떻게 측정할까? (금액, 효용 단위, 만족도 점수)

### 2. 확률 추정
다음을 통해 추정을 돕습니다:
- 과거 데이터 (가용한 경우)
- 유사 상황 비교
- 전문가 판단 (사용자 경험)
- 주관적 평가 (데이터 없는 경우)

### 3. 시각화
마크다운으로 트리를 그립니다:

```
결정
├─ 옵션 A
│  ├─ 결과 A1 (X%) → 가치 Y
│  └─ 결과 A2 (Z%) → 가치 W
└─ 옵션 B
   └─ 결과 B1 (100%) → 가치 V
```

### 4. EV 계산
각 옵션에 대해:
```
EV_A = (X% x Y) + (Z% x W)
EV_B = V
```

### 5. 추천
EV가 가장 높은 옵션 = 합리적 최선의 선택.

**단, 맥락을 추가합니다:**
- 위험 감수 성향 (최악의 경우를 감당할 수 있는가)
- 시간 범위 (결과가 언제 필요한가)
- 기타 요소 (평판 리스크, 감정, 윤리)

## 적용 영역별 예시

### 트레이딩 및 투자
- **포지션 크기:** 자본의 5%, 10%, 20% 옵션 → 각각의 수익/손실 확률
- **진입 타이밍:** 지금 진입 / -5% 대기 / -10% 대기 → 기회비용 vs 진입가

### 비즈니스 전략
- **제품 출시:** 출시/미출시 → 성공/실패 → 매출, 시장점유율, 비용
- **채용 결정:** 후보 A / 후보 B / 미채용 → 온보딩 성공 / X개월 후 퇴사

### 개인 결정
- **커리어 전환:** 현직 유지 / 이직 / 창업 → 성공/실패 → 급여, 만족도, 성장
- **부동산:** 집 A / 집 B / 계속 임대 → 가격 상승/하락 → 순자산, 월비용

## 최종 체크리스트

추천 전에 확인:
- 모든 옵션이 포함되었는가
- 각 분기에서 확률 합계가 100%인가
- 가치가 현실적인가 (환상이 아닌가)
- 최악의 시나리오가 사용자에게 명확한가
- 위험/보상 비율이 명시적인가
- 방법론의 한계가 언급되었는가
- 정성적 맥락이 추가되었는가 (EV만이 아닌)

## 방법론의 한계

- **불안정성** — 데이터 변화가 작아도 트리가 크게 달라질 수 있음
- **주관성** — 확률 추정이 종종 "감"에 의존
- **단순화** — 현실은 모델보다 복잡
- **위험 선호 미반영** — 위험 중립을 가정

**그러나:** 이 방법의 가치는 **사고를 구조화**하는 것입니다. 숫자가 정확하지 않더라도, 모든 분기를 강제로 생각하게 만드는 프로세스가 중요합니다.
