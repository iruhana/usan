# Word 문서 (DOCX) 가이드

## 문서 구조

- DOCX는 XML 파일을 담은 ZIP 파일이다. `word/document.xml`이 본문, `word/styles.xml`이 스타일을 담고 있다.
- 텍스트는 Run(`<w:r>`) 단위로 분할된다. 각 Run은 균일한 서식을 가지며, 한 단어가 여러 Run에 걸칠 수 있다.
- 단락(`<w:p>`)은 Run을 포함한다. 하나의 단락이 하나의 텍스트 블록이라고 가정하면 안 된다.
- 섹션(Section)은 페이지 레이아웃을 제어한다. 머리글/바닥글, 여백, 방향은 섹션별로 설정된다.

## 스타일 vs 직접 서식

- 스타일(제목 1, 본문 등)은 이름이 있고 재사용 가능하다. 직접 서식은 인라인이며 스타일을 덮어쓴다.
- 직접 서식을 제거하면 기본 스타일이 나타난다. 정리 작업에 유용하다.
- 문자 스타일은 Run에, 단락 스타일은 단락에 적용된다. 둘은 레이어처럼 중첩된다.
- 연결 스타일(Linked style)은 둘 다 가능하다. 단락 전체에 적용할 때와 선택 텍스트에 적용할 때 동작이 다르다.

## 목록과 번호 매기기

- 번호 매기기는 복잡하다: `abstractNum`이 패턴을 정의하고, `num`이 이를 참조하며, 단락이 `numId`를 참조한다.
- 번호 재시작은 자동이 아니다. 명시적으로 `<w:numPr>`에 재시작 플래그를 설정해야 한다.
- 글머리 기호와 번호 목록은 같은 번호 매기기 시스템을 공유한다. 둘 다 `numId`를 사용한다.
- 들여쓰기는 번호 매기기와 별도로 제어된다. 목록이 시각적 들여쓰기 없이 존재할 수 있다.

## 머리글, 바닥글, 섹션

- 각 섹션은 서로 다른 머리글/바닥글을 가질 수 있다. 첫 페이지, 홀수 페이지, 짝수 페이지별로 설정 가능하다.
- 섹션 나누기: 다음 페이지, 연속, 짝수/홀수 페이지가 있다. 페이지 매김에 영향을 준다.
- 머리글/바닥글은 별도 XML 파일에 저장된다. 섹션 속성에서 참조한다.
- 페이지 번호는 필드(field)이며 정적 텍스트가 아니다. 열기 또는 인쇄 시 업데이트된다.

## 변경 내용 추적과 메모

- 변경 내용 추적은 원본과 수정본을 같은 문서에 저장한다. 최종 확정하려면 수락/거부해야 한다.
- 삭제된 텍스트는 `<w:del>` 래퍼와 함께 여전히 존재한다. 보이는 것이 전체 내용이라 가정하면 안 된다.
- 메모(Comment)는 북마크 ID를 통해 범위를 참조한다. `<w:commentRangeStart>`에서 `<w:commentRangeEnd>`까지이다.
- 수정 ID는 누가 무엇을 변경했는지 추적한다. 수락 후에도 메타데이터가 남아 있다.

## 필드와 동적 콘텐츠

- 필드는 코드와 캐시된 결과를 가진다. `{ DATE \@ "yyyy-MM-dd" }` vs 표시되는 날짜.
- 목차(TOC), 페이지 번호, 상호 참조는 필드이다. 필드를 업데이트해야 새로고침된다.
- 하이퍼링크는 필드이거나 직접 `<w:hyperlink>`일 수 있다. 둘 다 유효하다.
- MERGEFIELD는 편지 병합용이다. 병합 실행 전까지는 자리표시자이다.

## 호환성

- 호환 모드는 이전 Word 버전의 기능으로 제한한다. `w:compat` 설정을 확인하라.
- LibreOffice/Google Docs: 복잡한 서식이 어긋날 수 있다. 왕복(roundtrip) 테스트 필요.
- 임베디드 폰트가 전달되지 않을 수 있다. 대체 폰트가 사용된다.
- DOCM은 매크로 포함(보안 위험), DOC는 레거시 바이너리 형식이다.

## 흔한 실수

- 빈 단락으로 간격 조절 -- 단락 스타일의 위/아래 간격을 사용하는 것이 바람직하다.
- 단락 내 수동 페이지 나누기 -- 레이아웃 제어에는 섹션 나누기를 사용하라.
- 머리글 내 이미지: 관계 ID는 파트별이다. 같은 이미지라도 머리글에 별도 관계가 필요하다.
- 복사-붙여넣기 시 원본 스타일이 따라온다. 스타일 갤러리에 중복이 생길 수 있다.
