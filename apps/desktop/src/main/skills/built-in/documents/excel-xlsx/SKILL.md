# Excel 스프레드시트 (XLSX) 가이드

## 활용 시점

사용자가 Excel 파일(.xlsx, .xls, .xlsm)을 읽거나, 쓰거나, 생성해야 할 때 사용한다. 데이터 타입 변환, 날짜 직렬화, 수식 평가, 크로스 플랫폼 이슈를 처리한다.

## 핵심 규칙

### 1. 날짜는 일련 번호다
Excel은 날짜를 1900-01-01(Windows) 또는 1904-01-01(Mac 레거시) 이후 일수로 저장한다. 변환 전에 통합 문서의 날짜 시스템을 확인하라. 시간은 소수점: 0.5 = 정오, 0.25 = 오전 6시.

### 2. 1900년 윤년 버그
Excel은 1900년을 윤년으로 잘못 처리한다. 일련 번호 60이 1900년 2월 29일(존재하지 않는 날짜)을 나타낸다. 1900년 3월 1일 이전 날짜 계산 시 이 점을 고려하라.

### 3. 15자리 정밀도 제한
15자리를 초과하는 숫자는 조용히 잘린다. 다음은 반드시 TEXT 형식으로 저장하라: 전화번호, ID, 신용카드 번호, 긴 숫자 식별자. 앞에 0이 붙는 값도 TEXT가 필요하다.

### 4. 수식 vs 캐시된 값
셀에 수식과 캐시된 결과가 모두 있을 수 있다. 일부 리더는 수식 문자열을, 다른 리더는 캐시된 값을 반환한다. 캐시된 값이 오래되었을 수 있으면 재계산을 강제하라.

### 5. 셀 병합은 함정이다
병합된 범위의 왼쪽 상단 셀만 값을 보유한다. 병합 내 다른 셀을 읽으면 비어 있다. 숨겨진 행/열에도 데이터가 있다.

### 6. 크로스 플랫폼 테스트 필수
Windows와 Mac Excel은 날짜 시스템이 다를 수 있다. LibreOffice/Google Sheets는 모든 기능을 지원하지 않을 수 있다. 소비자가 불분명할 때는 항상 왕복 호환성을 테스트하라.

### 7. 대용량 파일은 스트리밍 사용
대용량 파일을 전부 RAM에 로드하면 메모리 문제가 발생한다. 10만 행 이상의 파일에는 스트리밍 리더(행 단위)를 사용하라. 일부 라이터는 끝에 빈 행을 패딩할 수 있다.

## 흔한 함정

- **읽기 시 타입 추론** -- 텍스트로 저장된 숫자는 텍스트로 남는다. 명시적 변환이 필요하다.
- **열 인덱스 혼동** -- A=0인지 A=1인지 라이브러리마다 다르다. 항상 규칙을 확인하라.
- **셀 내 줄바꿈** -- `\n`은 동작하지만 셀에 "텍스트 줄 바꿈" 서식이 필요하다.
- **외부 참조** -- `[Book.xlsx]Sheet!A1`은 원본 파일 이동 시 깨진다.
- **비밀번호 보호** -- 해제가 쉽다. 진정한 보안이 아니다. 필요 시 외부에서 파일을 암호화하라.
- **XLSM 파일** -- 매크로 포함(보안 위험). XLSB는 바이너리(빠르지만 호환성 낮음).
- **공유 문자열** -- 대용량 파일은 텍스트 인덱스를 재사용한다. 라이브러리가 처리하지만 알아두어야 한다.

## 형식 제한

| 형식 | 행 | 열 | 비고 |
|------|-----|---------|------|
| XLSX | 1,048,576 | 16,384 (XFD) | 현대 기본값 |
| XLS | 65,536 | 256 | 레거시, 지양 |
| CSV | 무제한 | 무제한 | 서식 없음 |

## 보안 및 개인정보

- 모든 파일 처리는 로컬에서 수행된다.
- 외부 서비스를 호출하지 않는다.
- 네트워크 접근이 필요하지 않다.
