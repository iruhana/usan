# 딥 리서치

다중 웹 소스에서 철저하고 출처가 포함된 리포트를 생성하는 심층 리서치 스킬입니다. 유료 API 불필요 -- DuckDuckGo 검색을 사용합니다.

## 사전 요구사항

- DuckDuckGo 검색 도구 (web_search)
- curl (페이지 전문 조회용)
- API 키 불필요

## 절차

사용자가 특정 주제에 대한 리서치를 요청하면 다음 워크플로우를 따릅니다:

### 1단계: 목표 파악 (30초)

간단한 확인 질문 1-2개:
- "목적이 학습, 의사결정, 글쓰기 중 어느 쪽인가요?"
- "특정 관점이나 깊이가 필요한가요?"

사용자가 "그냥 리서치해줘"라고 하면 합리적인 기본값으로 바로 진행합니다.

### 2단계: 리서치 계획 (검색 전 구상)

주제를 3-5개 하위 질문으로 분해합니다. 예시:
- 주제: "AI가 의료에 미치는 영향"
  - 현재 의료 분야의 주요 AI 응용은?
  - 측정된 임상 결과는?
  - 규제 과제는?
  - 이 분야를 선도하는 기업은?
  - 시장 규모와 성장 궤적은?

### 3단계: 다중 소스 검색 실행

각 하위 질문에 대해 웹 검색을 수행합니다:

```
# 웹 검색
web_search("<하위 질문 키워드>")

# 뉴스 검색 (시사 주제인 경우)
web_search("<주제> 뉴스")
```

**검색 전략:**
- 하위 질문당 2-3가지 키워드 변형 사용
- 웹 + 뉴스 검색 혼합
- 총 15-30개 고유 소스 확보 목표
- 우선순위: 학술 > 공식 > 신뢰할 수 있는 뉴스 > 블로그 > 포럼

### 4단계: 주요 소스 심층 읽기

가장 유망한 URL에 대해 전체 콘텐츠를 조회합니다:

```bash
curl -sL "<url>" | python3 -c "
import sys, re
html = sys.stdin.read()
text = re.sub('<[^>]+>', ' ', html)
text = re.sub(r'\s+', ' ', text).strip()
print(text[:5000])
"
```

깊이를 위해 3-5개 주요 소스를 전문 읽기합니다. 검색 스니펫에만 의존하지 않습니다.

### 5단계: 종합 및 리포트 작성

다음 구조로 리포트를 작성합니다:

```markdown
# [주제]: 심층 리서치 리포트
*생성일: [날짜] | 소스: [N]개 | 신뢰도: [높음/보통/낮음]*

## 요약
[핵심 발견사항 3-5문장 개요]

## 1. [첫 번째 주요 테마]
[인라인 출처가 포함된 발견사항]
- 핵심 포인트 ([출처명](url))
- 뒷받침 데이터 ([출처명](url))

## 2. [두 번째 주요 테마]
...

## 3. [세 번째 주요 테마]
...

## 핵심 시사점
- [실행 가능한 인사이트 1]
- [실행 가능한 인사이트 2]
- [실행 가능한 인사이트 3]

## 출처
1. [제목](url) -- [한 줄 요약]
2. ...

## 방법론
[N]개 쿼리로 웹 및 뉴스 검색. [M]개 소스 분석.
조사한 하위 질문: [목록]
```

### 6단계: 저장 및 전달

리포트를 저장합니다:
```bash
mkdir -p ~/usan/research/[slug]
# 리포트를 ~/usan/research/[slug]/report.md 에 저장
```

전달 방법:
- **짧은 주제**: 전체 리포트를 채팅에 게시
- **긴 리포트**: 요약 + 핵심 시사점을 게시하고, 전체 리포트는 파일로 제공

## 품질 규칙

1. **모든 주장에는 출처가 필요합니다.** 출처 없는 주장은 금지.
2. **교차 검증.** 하나의 소스만 언급하면 미확인으로 표시.
3. **최신성 중요.** 최근 12개월 이내 소스 우선.
4. **격차 인정.** 하위 질문에 대한 정보를 찾지 못하면 솔직히 기재.
5. **환각 금지.** 모르면 "충분한 데이터를 찾지 못했습니다"라고 명시.

## 사용 예시

```
"핵융합 에너지의 현재 상태를 리서치해줘"
"2026년 백엔드 서비스에서 Rust vs Go 심층 비교"
"SaaS 부트스트래핑 전략 조사해줘"
"현재 한국 부동산 시장 상황은?"
```
