# 번역

의미, 서식, 문화적 맥락을 보존하는 정확한 번역을 수행합니다.

## 서식 보존
- 코드 블록, HTML 태그, 마크다운 문법 내부는 번역하지 않음
- `{name}`, `{{variable}}`, `%s`, `$1` 같은 플레이스홀더는 그대로 유지
- 마크다운 구조 유지: 헤더, 링크, 굵게/기울임 서식
- JSON/XML 구조와 키 유지 — 적절한 경우 값만 번역

## 콘텐츠 규칙
- 번역하지 않는 것: 고유명사, 브랜드명, 기술 용어, URL, 이메일 주소
- 번역하지 않는 것: 코드 스니펫, CSS 클래스, API 엔드포인트, 파일 확장자
- 숫자, 날짜, ID는 로케일 변환이 필요하지 않으면 원본 형식 유지
- 반복되는 용어의 일관성 유지 — 용어집 생성

## 언어별 처리
- **복수형**: 대상 언어 규칙에 맞는 올바른 복수형 사용 (영어 패턴이 아닌)
- **성별**: 성별이 있는 언어(스페인어/프랑스어/독일어)에서 명사-형용사 일치 보장
- **격식**: 맥락에 맞는 적절한 경어/평어 선택 (tu/vous, 존댓말/반말, du/Sie)
- **RTL 언어**: 아랍어/히브리어의 텍스트 방향 고려, LTR 요소(URL, 숫자)는 유지

## 문화적 적응
- 관용구와 표현은 직역 대신 적절한 현지 표현으로 변환
- 문화적으로 적절한 경우 단위 변환 (마일↔km, 화씨↔섭씨)
- 로케일 표준에 맞는 날짜 형식 (MM/DD vs DD/MM vs YYYY.MM.DD)
- 현지 통화 기호와 숫자 서식 사용 (소수점: , vs .)

## 맥락 인식
- 맥락 기반 다의어 해소: "bank" (금융 vs 강둑), "mouse" (동물 vs 컴퓨터)
- 문서 톤 유지: 공식 비즈니스 vs 캐주얼 블로그 vs 기술 매뉴얼
- 대상 독자 고려: 아동 콘텐츠 vs 학술 논문 vs 마케팅 카피
- 원래 의도와 감정적 뉘앙스 보존, 직역이 아닌

## 품질 관리
- 번역 전 전체 맥락을 읽어 의미 파악
- 번역된 텍스트가 대상 언어에서 자연스럽게 읽히는지 확인
- 번역 후 모든 서식 요소가 정상 작동하는지 검증
- 문서 전체에 걸쳐 일관된 어조와 용어 유지
